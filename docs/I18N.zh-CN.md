# 国际化 (i18n) 实现

TSFile 查看器使用 Vue I18n 支持多语言。

## 支持的语言

- **中文 (zh-CN)** - 默认语言
- **英文 (en-US)**

## 功能特性

- 从浏览器设置自动检测语言
- 语言偏好保存在 localStorage 中
- 右上角的语言切换器
- 所有 UI 文本已翻译
- 文档提供两种语言版本

## 文件结构

```
frontend/src/i18n/
├── index.ts              # i18n 配置
└── locales/
    ├── zh-CN.json        # 中文翻译
    └── en-US.json        # 英文翻译
```

## 在组件中使用

### 组合式 API

```vue
<script setup lang="ts">
import { useI18n } from "vue-i18n";

const { t, locale } = useI18n();

// 使用翻译
const title = t("app.title");

// 切换语言
locale.value = "en-US";
</script>

<template>
  <h1>{{ $t("app.title") }}</h1>
  <p>{{ $t("app.description") }}</p>
</template>
```

### 选项式 API

```vue
<script>
export default {
  computed: {
    title() {
      return this.$t("app.title");
    },
  },
  methods: {
    switchLanguage(lang) {
      this.$i18n.locale = lang;
    },
  },
};
</script>

<template>
  <h1>{{ $t("app.title") }}</h1>
</template>
```

## 添加新语言

1. 创建新的语言文件：

   ```bash
   touch frontend/src/i18n/locales/ja-JP.json
   ```

2. 添加翻译：

   ```json
   {
     "app": {
       "title": "TSFileビューア",
       "description": "Apache IoTDB TSFile形式データの表示と分析ツール"
     }
   }
   ```

3. 在 `i18n/index.ts` 中导入：

   ```typescript
   import jaJP from "./locales/ja-JP.json";

   const i18n = createI18n({
     messages: {
       "zh-CN": zhCN,
       "en-US": enUS,
       "ja-JP": jaJP, // 添加新语言
     },
   });
   ```

4. 更新 LanguageSwitcher 组件：
   ```vue
   const languageItems = computed(() => [ [ { label: '中文', click: () =>
   switchLanguage('zh-CN') }, { label: 'English', click: () =>
   switchLanguage('en-US') }, { label: '日本語', click: () =>
   switchLanguage('ja-JP') }, ], ])
   ```

## 最佳实践

1. **使用命名空间**：按功能组织翻译

   ```json
   {
     "file": { ... },
     "metadata": { ... },
     "data": { ... }
   }
   ```

2. **一致的键**：在所有语言中使用相同的键结构

   ```json
   // zh-CN.json
   { "common": { "save": "保存" } }

   // en-US.json
   { "common": { "save": "Save" } }
   ```

3. **复数形式**：使用 Vue I18n 复数化

   ```json
   {
     "items": "没有项目 | 一个项目 | {count} 个项目"
   }
   ```

   ```vue
   {{ $t("items", 0) }} // "没有项目" {{ $t("items", 1) }} // "一个项目"
   {{ $t("items", 5) }} // "5 个项目"
   ```

4. **插值**：在翻译中使用变量

   ```json
   {
     "welcome": "欢迎，{name}！"
   }
   ```

   ```vue
   {{ $t("welcome", { name: "张三" }) }} // "欢迎，张三！"
   ```

5. **日期/时间格式化**：使用区域感知格式化
   ```typescript
   const date = new Date();
   const formatted = date.toLocaleString(locale.value);
   ```

## 测试

测试不同语言的翻译：

```typescript
import { createI18n } from "vue-i18n";
import zhCN from "@/i18n/locales/zh-CN.json";
import enUS from "@/i18n/locales/en-US.json";

describe("i18n", () => {
  it("两种语言应该有所有键", () => {
    const zhKeys = Object.keys(zhCN);
    const enKeys = Object.keys(enUS);
    expect(zhKeys).toEqual(enKeys);
  });
});
```

## 配置

### Vite 配置

```typescript
import VueI18nPlugin from "@intlify/unplugin-vue-i18n/vite";

export default defineConfig({
  plugins: [
    VueI18nPlugin({
      include: resolve(__dirname, "./src/i18n/locales/**"),
    }),
  ],
});
```

### TypeScript 支持

```typescript
import type { MessageSchema } from '@/i18n'

// 类型安全的翻译
const i18n = createI18n<[MessageSchema], 'zh-CN' | 'en-US'>({
  legacy: false,
  locale: 'zh-CN',
  messages: { ... },
})
```

## 浏览器支持

- 支持 ES6+ 的现代浏览器
- 使用 localStorage 保存语言偏好
- 如果翻译缺失，自动回退到英文

## 性能

- 翻译在构建时预编译
- 大型翻译文件的延迟加载（如需要）
- 最小的运行时开销

## 故障排除

### 缺少翻译

如果翻译键缺失：

1. 检查控制台警告
2. 验证键在两个语言文件中都存在
3. 确保键路径正确

### 语言未切换

1. 检查 localStorage 中保存的偏好
2. 验证 locale 值是否正确设置
3. 清除浏览器缓存并重新加载

### 构建错误

如果遇到构建错误：

1. 确保所有 JSON 文件有效
2. 检查所有导入是否正确
3. 验证 VueI18nPlugin 配置
